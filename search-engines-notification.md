# Уведомление поисковых систем о смене домена

После замены всех канонических URL с `вашдомен.ru` на `escort-bar.live`, необходимо уведомить поисковые системы о смене доменного имени, чтобы они могли обновить свои индексы.

## Уведомление Яндекса через Яндекс.Вебмастер

1. Войдите в [Яндекс.Вебмастер](https://webmaster.yandex.ru/).
2. Убедитесь, что оба домена (старый `вашдомен.ru` и новый `escort-bar.live`) подтверждены в Вебмастере.
3. Перейдите в раздел "Инструменты" → "Переезд сайта".
4. Укажите, что старый домен `вашдомен.ru` переехал на новый `escort-bar.live`.
5. Заполните все необходимые поля и следуйте инструкциям Яндекса.
6. После успешной настройки переезда, Яндекс начнет обрабатывать редиректы с одного домена на другой.

## Уведомление Google через Search Console

1. Войдите в [Google Search Console](https://search.google.com/search-console).
2. Убедитесь, что оба домена (старый и новый) подтверждены в Search Console.
3. Перейдите в раздел "Настройки" → "Изменение адреса".
4. Укажите, что старый домен `вашдомен.ru` переехал на новый `escort-bar.live`.
5. Следуйте инструкциям Google для завершения процесса переезда.
6. После успешной настройки, Google будет учитывать изменение адреса при индексации.

## Настройка 301 редиректов на сервере

Важно настроить 301 редиректы со старого домена на новый. Это можно сделать в конфигурации Nginx:

```nginx
server {
    listen 80;
    server_name xn--80adhe8ahe2f.ru вашдомен.ru;
    
    # Редирект всех запросов на новый домен с сохранением пути
    return 301 https://escort-bar.live$request_uri;
}
```

После настройки редиректов, проверьте их работу, посетив старый домен - должно автоматически перенаправить на новый.

## Проверка индексации с помощью IndexNow

1. Отправьте уведомление через [IndexNow](https://www.indexnow.org/), чтобы ускорить индексацию нового домена.
2. Запустите скрипт для отправки всех URL сайта:

```bash
node backend/scripts/indexNowSubmit.js
```

## Мониторинг процесса переезда

1. Регулярно проверяйте статус в Яндекс.Вебмастер и Google Search Console.
2. Отслеживайте количество страниц, проиндексированных по новому домену.
3. Проверяйте, что количество страниц в индексе старого домена уменьшается.
4. Мониторьте трафик и убедитесь, что он корректно перенаправляется с старого домена на новый.

## Обновление внешних ссылок

По возможности, обновите все внешние ссылки на ваш сайт, чтобы они вели на новый домен `escort-bar.live`. Это касается:

1. Профилей в социальных сетях
2. Каталогов и справочников
3. Рекламных кампаний
4. Партнерских ссылок

Это ускорит процесс миграции и поможет избежать потери трафика.
